Vision-Guided UR3 Robotic Drawing System

A ROS-based robotic drawing system that converts a captured selfie into vectorized contours and reproduces it on paper using a UR3 robot arm — from live image to physical drawing.

Python ROS OpenCV MoveIt Inverse Kinematics RGB-D Camera Jan 2024 — Jun 2024

Problem

Transform a captured selfie into a drawable path and accurately reproduce it on paper using a robotic arm.

My Role

Image processing pipeline, UR3 motion control, paper detection, and 3D coordinate transformation.

Outcome

End-to-end robotic drawing pipeline with sub-2mm drawing error and smooth trajectory execution.

System Workflow

The system captures a selfie, removes the background, extracts facial contours, vectorizes the edges, optimizes the drawing path via TSP heuristics, maps the coordinate frame to the physical paper, and executes the path using UR3 robotic control.

Image Processing Pipeline

Developed a complete image processing pipeline: background removal, grayscale conversion, facial landmark detection, edge extraction, contour processing, and vectorization. Three interpolation methods were evaluated — nearest neighbour, bilinear, and bicubic — with bicubic selected for its path smoothness.

  • Background removal via rembg / depth thresholding
  • Canny edge detection + contour hierarchy analysis
  • Vectorized stroke generation from pixel contours
  • TSP heuristic path optimization — 40% reduction in redundant movement

Paper Detection & Coordinate Mapping

Mounted an Intel RealSense RGB-D camera on the UR3 end effector to detect paper corners in real time using colour and depth segmentation. Local camera coordinates were transformed into the UR3 global frame via a calibrated homography, ensuring the drawing was correctly aligned and scaled to the paper surface regardless of placement.

Robot Motion Control

Implemented smooth UR3 trajectory execution using inverse kinematics, MoveIt path planning, and joint trajectory monitoring. Pen length compensation was incorporated into the tool frame to maintain consistent contact force and line quality throughout the drawing.

  • MoveIt motion planning with collision avoidance
  • Real-time IK with joint limit monitoring
  • Tool centre point (TCP) calibration for pen tip offset
  • Drawing error < 2 mm across full paper area

Final Result

Successfully integrated perception, coordinate transformation, path planning, and robotic execution into a working physical system capable of drawing vectorized selfie portraits on paper.
